Markdown reference

Note bodies render with GitHub-Flavored Markdown. Type the syntax in the left column, see the rendered output on the right.

Headings

One # through six ######. The note title above the body is the page heading, so most note bodies start at ##.

Source
# Largest heading
## Section heading
### Subsection
#### Smaller subsection

Emphasis

Bold, italic, and strikethrough. Use sparingly.

Source
**Bold** for emphasis.
*Italic* for titles or terms.
***Both*** for stronger weight.
~~Strikethrough~~ for crossed-out text.
Rendered

Bold for emphasis.
Italic for titles or terms.
Both for stronger weight.
Strikethrough for crossed-out text.

Lists

Hyphen for unordered, number for ordered. Indent two spaces to nest.

Source
- First contact
- Second contact
  - Sub-detail
  - Another sub-detail
- Third contact

1. First step
2. Second step
3. Third step
Rendered

  • Second contact

    • Another sub-detail

Task lists

Checkboxes you can check off in the source.

Source
- [x] Found call number
- [x] Emailed Jianye He
- [ ] Visit library in person
- [ ] Photograph relevant pages

Blockquotes

For quoting a source, a translation, or a passage.

Source
> "終子二00一年三月廿二日"
> Passed away on March 22, 2001.

The character 終 here is classical/literary.
Rendered

"終子二00一年三月廿二日"
Passed away on March 22, 2001.

Code

Backticks for inline code; triple-backtick fenced blocks for longer snippets, transcribed inscriptions, or tabular data that's easier as plain text.

Line breaks

A blank line between paragraphs. For a hard break inside a paragraph, end the line with two spaces or a backslash.

Second paragraph.

Line one\
Line two

Mixing Chinese and English inline

Just type. CJK and Latin characters render side by side without escaping. Markdown formatting works around them normally.

Source
**Goal:** Find article featuring 陳阿鴻 (Chan, Ah Hung) in *Jinshan Shibao* 金山時報, March 1940.
